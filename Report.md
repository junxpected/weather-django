## 1. Вибір фреймворку

Для розробки веб-додатку було обрано фреймворк **Django** (Python).

**Причини вибору:**
- Вбудована ORM для роботи з базою даних
- Система шаблонів (Django Templates)
- Вбудований захист від CSRF, SQL-ін'єкцій та XSS
- Простота розгортання та налагодження

---

## 2. Опис проєкту

**Weather App** — веб-додаток для перегляду погоди в реальному часі.

**Функціонал:**
- Пошук погоди за назвою міста
- Автоматичне визначення місцезнаходження через геолокацію
- Відображення поточної погоди (температура, вологість, вітер, відчувається як)
- Прогноз погоди на 5 днів
- Інтерактивна карта з позначенням міста (Leaflet.js)
- Автодоповнення при введенні міста
- Історія пошуку збережена в базі даних
- Glassmorphism дизайн з відео-фоном

## 4. Схема бази даних

Використовується одна модель — `SearchHistory`:

| Поле | Тип | Опис |
|------|-----|------|
| id | AutoField | Первинний ключ (автоматично) |
| city | CharField(100) | Назва міста |
| searched_at | DateTimeField | Час пошуку (автоматично) |

```python
class SearchHistory(models.Model):
    city = models.CharField(max_length=100)
    searched_at = models.DateTimeField(auto_now_add=True)

    class Meta:
        ordering = ['-searched_at']

    def __str__(self):
        return self.city
```

## 6. Бізнес-логіка

### Проксі до OpenWeather API
Фронтенд не звертається до OpenWeather напряму — всі запити йдуть через Django сервер. Це приховує API ключ від користувача.

```
Браузер → /api/weather/?city=Lviv → Django → OpenWeather API → Django → Браузер
```

### Геолокація
При завантаженні сторінки браузер запитує геолокацію користувача. Координати передаються на сервер і погода відображається автоматично.

### Збереження історії
Кожне унікальне місто зберігається в БД. При повторному пошуку того ж міста — дублікат не створюється.

## 7. Засоби безпеки

| Загроза | Захист |
|---------|--------|
| CSRF атаки | Django вбудований `CsrfViewMiddleware`, токен у кожному POST запиті |
| SQL-ін'єкції | Django ORM екранує всі запити автоматично |
| XSS | Django Templates екранують HTML символи |
| Витік API ключа | Ключ зберігається у `.env` файлі на сервері, не передається клієнту |

## Висновок

В ході практичного заняття було успішно розроблено веб-додаток Weather App з використанням фреймворку Django. Додаток реалізує повний стек MVT архітектури, включає роботу з базою даних через ORM, проксі до зовнішнього API, засоби безпеки та покриття юніт тестами. Порівняно з оригінальною JS версією, Django версія є більш безпечною, масштабованою та готовою до розгортання на сервері.
